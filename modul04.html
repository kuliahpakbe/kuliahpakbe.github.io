Protokol Administrasi Jarak Jauh (Remote)

Seorang administrator sistem jarang duduk di depan fisik server. Mereka bekerja dari laptop (Client) untuk mengendalikan server jarak jauh menggunakan protokol jaringan.

Telnet (Masa Lalu)

Protokol usang. Mengirimkan semua data komando dan password ke server dalam bentuk teks terang (plaintext). Sangat mudah disadap oleh hacker di jaringan yang sama.

Secure Shell / SSH (Standar Modern)

Membentuk "lorong terenkripsi" kriptografis antara laptop Anda dan server. Jika disadap, hacker hanya akan melihat data acak yang tidak bisa dibaca.

Sintaks: ssh [username]@[ip_address] -p [port]

Menaklukkan Anomali Sistem File

Dalam sistem Linux, nama file dengan karakter khusus (spasi, tanda strip/minus) akan membuat terminal kebingungan dan memicu error. Pelajari trik penanganannya.

Masalah Karakter Spasi

␣

Jika ada file bernama data rahasia.txt, mengetik cat data rahasia.txt akan membuat Bash mencari dua file berbeda: "data" dan "rahasia.txt".

Solusi Quoting:
cat "data rahasia.txt"
Solusi Escaping:
cat data\ rahasia.txt

Masalah Karakter Strip (-)

-

Jika ada file bernama -fileku, perintah cat -fileku akan memicu error karena Bash mengira -fileku adalah sebuah opsi (flags), bukan nama file.

Solusi Path Relatif:
cat ./-fileku

Penambahan ./ memaksa Bash membacanya sebagai alamat file di direktori saat ini.

Man-In-The-Middle (MITM)

Saat Anda menggunakan SSH untuk pertama kali, sistem akan memunculkan peringatan "The authenticity of host... can't be established" beserta sebuah kode Fingerprint acak.

Mengapa ini penting?

Ini adalah perlindungan terhadap MITM Attack. Hacker bisa membuat server palsu yang mencegat koneksi Anda untuk mencuri password. Fingerprint memastikan bahwa server yang merespons koneksi benar-benar server asli yang Anda tuju, bukan server penipu di tengah jalan.
