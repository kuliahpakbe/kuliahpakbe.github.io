Modul Praktikum 3

Manajemen User, Group, & Hak Akses

Pahami arsitektur Multi-User Linux. Kuasai prinsip Least Privilege, manipulasi kepemilikan file, dan pelajari cara menghitung matriks perizinan (Permissions) secara interaktif.

🧮 Coba Kalkulator Chmod

Konsep Dasar & Identitas Sistem

Linux memisahkan wewenang sistem secara ketat. Pahami perbedaan antara pengguna biasa, grup, dan peran absolut dari Superuser.

Superuser (Root)

Pengguna absolut dengan UID 0. Kebal terhadap semua aturan hak akses. Dapat memodifikasi atau menghapus file apa pun di seluruh sistem.

Regular User

Pengguna biasa. Hanya memiliki kekuasaan penuh di dalam direktori pribadinya sendiri (/home/username). Diatur ketat oleh permission.

Group

Kumpulan dari beberapa user. Memudahkan administrator memberi hak akses kolaborasi pada satu direktori kepada banyak orang sekaligus.

Berkas Identitas Vital

Linux tidak menyimpan data user di dalam database ajaib, melainkan di dalam dua file teks murni yang menjadi fondasi keamanan sistem:

  • /etc/passwd

    Berkas publik berisi pemetaan Username ke UID dan GID, letak direktori home, dan shell asal. Semua orang bisa membacanya.

  • /etc/shadow

    Berkas sangat rahasia penyimpan kata sandi yang telah dienkripsi (Hash). Hanya Root yang diizinkan membacanya.

# Contoh isi dari /etc/passwd
root:x:0:0:root:/root:/bin/bash
budi:x:1000:1000:Budi Santoso,,,:/home/budi:/bin/bash
# Kunci Format Pembacaan:
Username : Pwd_X : UID : GID : Info Tambahan : Home_Dir : Shell

Kalkulator Hak Akses (Chmod Oktal)

Setiap file memiliki 3 blok akses. Centang kotak R, W, X di bawah ini untuk melihat bagaimana Linux mengkalkulasi angka oktal dan mendistribusikan kekuasaan.

Matriks Izin (Permissions)

User (Pemilik) 6
Group (Rekan) 4
Others (Publik) 4

Hasil Eksekusi

644
-rw-r--r--

Keamanan Tingkat Lanjut

Skenario industri terkadang membutuhkan pengecualian dari aturan dasar RWX. Pelajari dua atribut khusus yang memodifikasi perilaku default Linux.

Sticky Bit (+t)

Proteksi Hapus di Folder Publik

Folder seperti /tmp memiliki izin Write untuk semua orang (777). Bahayanya, User A bisa iseng menghapus file milik User B.

Jika Sticky Bit aktif, hanya pemilik asli file tersebut (atau Root) yang bisa menghapus filenya, mengabaikan izin Write 777 yang ada di folder tersebut.

drwxrwxrwt 10 root root 4096 /tmp
chmod +t /nama_folder

SUID (+s)

Set-User-ID / Pendelegasian Kuasa

Program ganti password (passwd) perlu menulis ke file rahasia /etc/shadow. Bagaimana user biasa bisa mengubah sandinya sendiri tanpa diberi hak Root?

Jika file executable diberi flag SUID, program tersebut saat dijalankan akan beroperasi menggunakan hak akses si pemilik file tersebut (Root), bukan hak akses orang yang mengkliknya.

-rwsr-xr-x 1 root root 6820 /usr/bin/passwd
chmod u+s /path/program
