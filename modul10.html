Dasar-Dasar Shell Scripting

Tingkatkan level Anda dari Operator menjadi Administrator. Rangkai puluhan perintah Linux ke dalam satu berkas berkekuatan tinggi untuk mengotomatisasi tugas berulang yang membosankan.

Buka Debugger Skrip

Anatomi Skrip Bash

Sebuah *Shell Script* pada dasarnya adalah kumpulan perintah terminal biasa yang diketik dalam satu *file* teks, namun dilengkapi dengan kekuatan bahasa pemrograman seperti Variabel dan Logika Percabangan.

Shebang

Baris pertama yang wajib ada. Memberitahu OS program apa (interpreter) yang harus dipakai untuk membaca file ini.

#!/bin/bash

Variabel

Menyimpan data tanpa tipe yang kaku. Perhatikan tidak boleh ada spasi di sekitar tanda sama dengan (=).

NAMA="Budi"
echo "Halo $NAMA"

If / Else

Logika kondisional. Menggunakan tanda kurung siku `[ ]` untuk pengujian (test). Diakhiri dengan `fi`.

if [ $UMUR -gt 18 ]; then
  echo "Dewasa"
fi

Looping

Melakukan perulangan perintah (For / While). Dibuka dengan `do` dan ditutup dengan `done`.

for i in 1 2 3
do
  echo "Data $i"
done

Izin Eksekusi Skrip (Executable)

Setelah Anda menulis file skrip (contoh: backup.sh), file tersebut tidak akan bisa langsung dijalankan dengan perintah ./backup.sh. Sistem Linux secara default melindungi file dari eksekusi. Anda wajib memberinya izin eksekusi menggunakan perintah chmod +x backup.sh terlebih dahulu.

Analisis ROI Otomatisasi & Cron Jobs

Menulis skrip memang memakan waktu di awal, namun grafik di bawah ini membuktikan mengapa industri sangat bergantung pada DevOps dan otomatisasi untuk efisiensi jangka panjang.

Penjadwalan dengan Cron

Skrip sehebat apapun tidak berguna jika Anda masih harus bangun jam 3 pagi untuk menjalankannya. Linux memiliki daemon (servis latar belakang) bernama cron yang bertugas mengeksekusi skrip secara otomatis berdasarkan waktu.

# Format: Menit Jam Tanggal Bulan Hari Perintah
00 03 * * * /opt/backup.sh
-> "Jalankan backup.sh setiap jam 03:00 pagi setiap hari."
⚠️

Bahaya Human Error

Mengerjakan tugas administratif server (seperti membuat 100 *user* baru) secara manual rentan terhadap salah ketik (typo). Otomatisasi via skrip memastikan bahwa proses berjalan konsisten 100% setiap saat, dengan standar keamanan yang telah diaudit di dalam skrip.

Akumulasi Waktu Tersita: Manual vs Skrip (Per Bulan)

Asumsi: Tugas berulang memakan waktu 15 menit manual vs otomatisasi yang hanya butuh 2 jam pembuatan skrip di Hari ke-1.
