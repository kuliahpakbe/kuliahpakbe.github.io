Modul Praktikum 8

Version Control System dengan Git

Tinggalkan kebiasaan buruk menyimpan file dengan nama final_revisi_3_beneran.txt. Kuasai standar industri dalam melacak perubahan kode, memulihkan versi lama, dan berkolaborasi secara masif.

Latihan Perintah Git

Mengapa Kita Membutuhkan Git?

Version Control System (VCS) bertindak sebagai "Mesin Waktu" untuk proyek Anda. Tanpa VCS, kolaborasi kode adalah sebuah mimpi buruk.

Tradisi Konvensional (Tanpa Git)

Menyalin folder proyek berulang kali sebagai cadangan. Anda akhirnya memiliki deretan file membingungkan:

proyek_akhir/
├── web_ku_v1.html
├── web_ku_v2_revisi.html
└── web_ku_final_FIX_BANGET.html

Paradigma Git (Distributed VCS)

File di dalam folder proyek Anda hanya ada satu versi (yang sedang dikerjakan). Seluruh riwayat perubahan file tersebut direkam dan disimpan oleh Git di dalam folder tersembunyi bernama .git.

Git mencatat Siapa mengubah Apa dan Kapan.

Risiko Kehilangan Kode (Tanpa VCS)

Statistik umum penyebab hancurnya proyek mahasiswa atau tim.

Siklus Hidup File (Three States)

Ini adalah konsep paling penting di dalam Git. Sebuah file harus melewati tiga "ruangan" secara berurutan sebelum perubahan tersebut benar-benar terekam secara permanen.

Interactive Workflow

Klik tombol di bawah untuk memindahkan file index.html melewati tahapan Git.

Working Directory
Area kerja Anda. File baru dibuat atau dimodifikasi di sini. (Berstatus Untracked / Modified).
Staging Area (Index)
Ruang tunggu. Anda memilih file mana saja yang siap untuk direkam bersamaan.
Local Repository
Brankas sejarah. Perubahan direkam secara permanen sebagai satu "Commit".

Git CLI Simulator

Mari kita praktikkan alur kerja di atas! Konsol di bawah ini menyimulasikan repositori Git. Cobalah membuat file, melacak statusnya, dan melakukan komit pertama Anda.

Misi Latihan Praktikum

  1. Cek status awal: git status
  2. Inisialisasi repo: git init
  3. Buat sebuah file baru: touch tugas.txt
  4. Cek status (Penting!): git status
  5. Pindahkan ke Staging: git add tugas.txt
  6. Rekam ke Repository: git commit -m "Tugas pertama selesai"
  7. Cek riwayat sejarah: git log
* Klik pada kode untuk menyalin otomatis ke terminal.

Sinkronisasi Remote & GitHub

Repositori lokal di laptop Anda rentan hilang jika laptop rusak. Kita perlu menyinkronkannya ke server jauh (Remote Repository) seperti GitHub atau GitLab.

Alur Kerja Remote

Setelah kode direkam di Local Repository (Commit), Anda harus mendorongnya ke server GitHub agar bisa diakses oleh rekan tim.

# Menghubungkan folder lokal ke GitHub
git remote add origin https://github.com/budi/repo.git
# Mendorong (Upload) commit ke GitHub
git push -u origin main
# Menarik (Download) update dari rekan tim
git pull origin main
🔑

Keamanan: Personal Access Token (PAT)

Sejak tahun 2021, GitHub tidak lagi mengizinkan penggunaan password akun biasa saat Anda melakukan git push via terminal.

Solusi: Anda harus membuat Token Khusus (PAT) di menu pengaturan GitHub (Developer Settings). Token ini berupa deretan karakter acak yang panjang (misal: ghp_1a2b3c...).

Saat terminal meminta password untuk GitHub, tempelkan (Paste) token PAT Anda, BUKAN password akun web Anda!
