Mulai Simulasi Instalasi Pelajari Teori

Mengapa Virtualisasi?

Dalam paradigma bare-metal, 90% kapasitas server seringkali terbuang. Virtualisasi menciptakan ilusi perangkat keras untuk efisiensi maksimal.

Masalah Bare-Metal (Tradisional)

Sebuah OS dipasang langsung di atas hardware. Jika server hanya menjalankan satu aplikasi yang memakan 10% prosesor, 90% sisa komputasi terbuang sia-sia (underutilized).

Solusi Virtualisasi

Abstraksi logika memisahkan fisik dengan OS. Sebuah Host (Hardware) dipartisi menjadi beberapa "komputer" independen (Virtual Machine / Guest) oleh perangkat lunak menengah yang disebut Hypervisor.

Keunggulan Snapshot

Mengambil "foto" kondisi VM pada satu titik waktu. Jika OS rusak (Kernel Panic), administrator cukup me-restore snapshot untuk kembali normal dalam hitungan detik tanpa instal ulang.

Taksonomi & Arsitektur Hypervisor

Memahami perbedaan arsitektural antara Hypervisor Tipe 1 (Data Center) dan Tipe 2 (Laboratorium/Edukasi).

1

Hypervisor Tipe 1 (Bare-Metal)

Standar Industri / Cloud

Diinstal langsung di atas hardware tanpa OS perantara. Akses langsung ke hardware memberikan latensi rendah dan efisiensi maksimum.

Contoh: VMware ESXi, Proxmox VE
2

Hypervisor Tipe 2 (Hosted)

Edukasi / Development

Dijalankan sebagai aplikasi di dalam OS Host (misal Windows 11). Permintaan hardware dari VM diterjemahkan oleh OS Host terlebih dahulu.

Contoh: Oracle VirtualBox, VMware Workstation

Bantuan Masalah (Troubleshooting)

Kendala umum yang sering ditemui saat praktikum Modul 1 dan solusinya.

Penyebab & Solusi: Biasanya karena bug alokasi grafis. Matikan VM. Masuk ke VirtualBox Manager -> Settings -> Display. Ubah parameter Graphics Controller menjadi VMSVGA dan pastikan untuk menonaktifkan (uncheck) opsi Enable 3D Acceleration.

Penyebab & Solusi: Fitur virtualisasi perangkat keras belum diaktifkan di tingkat motherboard PC Host Anda. Lakukan Restart PC Anda, masuk ke BIOS/UEFI, cari menu Configuration/Advanced, dan aktifkan Intel Virtualization Technology (VT-x) atau AMD-V (SVM Mode).

Penyebab & Solusi: Alokasi "Base Memory" (RAM) yang Anda berikan pada VM terlalu kecil (di bawah 2 GB). Ubuntu 22.04 membutuhkan memori ekstra saat proses ekstraksi installer. Matikan paksa VM, naikkan alokasi RAM minimal ke 2048 MB, lalu ulangi proses instalasi.
