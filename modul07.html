Buka Simulator Paging

Memori Virtual & Page Fault

Sebuah program seringkali memiliki ukuran memori yang lebih besar dari RAM fisik yang tersedia. OS mengatasi hal ini dengan teknik Paging dan Virtual Memory.

Konsep Paging

OS memecah memori logis program menjadi blok-blok kecil berukuran sama yang disebut Pages. Sementara itu, RAM fisik dibagi menjadi blok-blok yang ukurannya persis sama disebut Frames.

Page Fault

Terjadi ketika CPU meminta sebuah Page untuk diproses, tetapi Page tersebut ternyata belum ada di dalam RAM (Frames). OS harus menghentikan proses sementara untuk mengambil Page tersebut dari Hardisk (Swap).

Page Hit

Terjadi ketika CPU meminta data, dan data tersebut sudah tersedia di dalam RAM. Ini adalah kondisi ideal karena proses eksekusi menjadi sangat cepat tanpa harus mengakses Hardisk.

Belady's Anomaly

"Bukankah menambah kapasitas memori (RAM/Frames) akan selalu mengurangi jumlah Page Fault?"

Fakta Industri: TIDAK SELALU! Pada algoritma FIFO, menambahkan jumlah frame justru bisa meningkatkan jumlah Page Fault. Fenomena aneh ini disebut Anomali Belady.

Sebaliknya, algoritma Stack-based seperti LRU dan Optimal kebal terhadap anomali ini.

🔥

Konsekuensi Thrashing

Jika RAM terlalu kecil dan proses membutuhkan banyak page, OS akan terus-menerus melakukan proses sibuk Swap In dan Swap Out dari hardisk tanpa henti. Kondisi di mana CPU menghabiskan lebih banyak waktu untuk mengurus antrean memori daripada mengeksekusi program itu sendiri disebut Thrashing, yang membuat komputer macet total (hang).
