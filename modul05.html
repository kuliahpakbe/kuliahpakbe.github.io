Anatomi Proses di Linux

Sebuah aplikasi hanya berupa barisan kode mati di hardisk sampai ia dipanggil oleh OS. Pahami perbedaan mendasarnya dan hierarki silsilah yang dibentuk.

💾

Program (Statis)

Baris kode biner yang tersimpan secara pasif di dalam disk (Hardisk/SSD). Program tidak mengonsumsi resource prosesor atau RAM aktif. (Contoh: file /usr/bin/nginx).

Dimuat ke Memori (Dieksekusi)
⚡

Proses (Dinamis)

Entitas yang hidup. Program yang sedang dimuat di RAM dan diinstruksikan kepada CPU. Setiap proses diberikan nomor identitas unik yang disebut PID (Process ID).

Process Tree (Silsilah Parent-Child)

Linux memulai segalanya dari satu leluhur. Jika proses Parent dibunuh, proses Child juga akan terpengaruh.

▶ systemd (PID 1) - The Mother of All Processes
▶ sshd (PID 850) // Daemon SSH
▶ sshd (PID 1400) // Sesi Budi
▶ bash (PID 1401) // Terminal Budi
▶ ps aux (PID 1450) // Perintah saat ini
▶ nginx (PID 900) // Web Server

Pengendalian & Sinyal Kematian (IPC)

Terminal Linux berjalan sinkron (menunggu perintah selesai). Jika ada proses macet atau berjalan terlalu lama, Anda harus mengendalikannya (Job Control) atau mengirimkan Sinyal Terminasi (Signals).

Job Control

Saat Anda menjalankan proses berat (contoh: dd if=/dev/zero...), terminal akan terkunci. Anda tidak bisa mengetik.

  • Ctrl + Z (Suspend): Menidurkan paksa proses tersebut. Terminal kembali bisa diketik.
  • bg: Membangunkan proses yang tertidur dan melemparnya agar berjalan diam-diam di Latar Belakang (Background).

Signals (kill)

Perintah kill bukan sekadar membunuh, melainkan mengirim Sinyal Komunikasi Antar-Proses (IPC).

  • SIGTERM (15): Kematian Elegan. "Tolong bersihkan memori dan simpan datamu, lalu berhentilah." (Aman untuk database).
  • SIGKILL (9): Kematian Mutlak Instan. Dihabisi langsung oleh Kernel. Transaksi terputus seketika. (Digunakan via kill -9 PID jika aplikasi macet total).

Analisis Observasi Kinerja (Evaluasi Lanjut)

Memahami luaran teknis dari alat monitoring untuk menyimpulkan kesehatan infrastruktur server, sesuai standar industri IT.

Zombie Process (Z)

Proses yang sudah selesai dieksekusi (mati), tetapi masih memiliki entri di tabel proses karena Parent-nya belum sempat membaca status kematiannya (reaping).

Analisis: Anda tidak bisa membunuh Zombie dengan kill -9 karena ia sudah mati. Solusinya: bunuh Parent prosesnya, atau tunggu systemd mengambil alih dan membersihkannya.

Load Average

Angka (contoh: 3.50, 1.20, 0.50) mewakili rata-rata antrean proses pada CPU dalam 1, 5, dan 15 menit terakhir.

Analisis: Pada server 1-Core, nilai 1.00 berarti CPU 100% penuh tanpa antrean. Nilai 3.50 berarti ada 2.5 proses yang harus antre menunggu giliran. Jika Load > jumlah Core, server dipastikan lagging (kelebihan beban).

I/O Bottleneck (vmstat)

Perintah vmstat menyoroti performa input/output memori dan disk. Kolom krusial adalah bi (Blocks In) dan bo (Blocks Out).

Analisis: Jika nilai bi/bo sangat tinggi secara konstan, sementara CPU bebas (idle), ini mengindikasikan Disk I/O Bottleneck. Sistem melambat bukan karena prosesor, melainkan karena hardisk lambat membaca/menulis data.
