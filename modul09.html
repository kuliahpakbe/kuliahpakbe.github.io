Modul Praktikum 9

Pengantar Ubuntu Server & Administrasi Jarak Jauh

Tinggalkan kenyamanan antarmuka grafis. Pelajari arsitektur murni sebuah peladen (Server), konfigurasikan IP Statis menggunakan standar Netplan (YAML), dan hubungkan diri Anda melalui terowongan Secure Shell (SSH).

Latihan Konfigurasi Netplan

Arsitektur Headless: Mengapa Tanpa GUI?

Sistem Operasi untuk Server di industri hampir tidak pernah diinstal bersamaan dengan Desktop Environment (seperti GNOME atau KDE). Arsitektur tanpa monitor fisik dan tanpa grafis ini disebut Headless.

Lingkungan Desktop (GUI)

  • Konsumsi Memori: Sangat boros. GUI (GNOME) membutuhkan minimal 1 GB hingga 2 GB RAM hanya untuk menampilkan antarmuka.
  • Keamanan: Memiliki jutaan baris kode tambahan (untuk *browser*, *media player*, animasi). Semakin banyak kode, semakin banyak celah eksploitasi (*Attack Surface* yang luas).
  • Fungsi: Dirancang untuk komputasi personal dan kenyamanan navigasi harian.

Lingkungan Server (Headless)

  • Konsumsi Memori: Sangat efisien. Saat *idle*, Ubuntu Server hanya mengonsumsi sekitar 150 MB hingga 200 MB RAM. Sisa RAM bisa difokuskan untuk *Database* atau *Web Server*.
  • Keamanan: *Attack Surface* sangat minimalis. Hanya aplikasi penting (seperti SSH) yang berjalan.
  • Fungsi: Dirancang untuk stabilitas, kinerja komputasi murni, dan dikendalikan 100% dari jarak jauh via CLI.

Konfigurasi Jaringan: Netplan & YAML

Pada Ubuntu modern (sejak versi 18.04), pengaturan alamat IP tidak lagi menggunakan berkas teks biasa, melainkan menggunakan utilitas Netplan dengan format bahasa YAML (*YAML Ain't Markup Language*).

Peringatan Kritis Sintaks YAML

YAML sangat ketat dan sensitif terhadap indentasi (lekukan). Anda TIDAK BOLEH menggunakan tombol TAB untuk memberikan jarak. Anda diwajibkan menggunakan ketukan tombol SPASI (biasanya kelipatan 2 spasi per anak hierarki). Satu spasi yang salah akan membuat seluruh koneksi *server* mati.

Misi Simulator: Atur IP Statis

Skrip di bawah ini memiliki kesalahan (memakai DHCP padahal server butuh IP Statis) dan struktur indentasi-nya sedikit berantakan. Tugas Anda:
1. Matikan DHCP (ubah true menjadi false atau no).
2. Pastikan susunan hirarkinya benar menggunakan Spasi (bukan TAB).
3. Klik tombol "netplan apply" untuk memvalidasi.

Alur Akses Jarak Jauh (SSH Workflow)

Setelah Server memiliki IP Statis yang kita konfigurasi di atas, Administrator tidak lagi perlu melihat layar *server*. Semua pekerjaan dilakukan dari Terminal Laptop (Host) menggunakan protokol OpenSSH.

1

Instalasi di Server

Secara *default*, layanan SSH mungkin belum berjalan. Kita harus menginstalnya di sisi Ubuntu Server.

sudo apt install openssh-server
2

Pastikan Layanan Aktif

Pastikan *daemon* (layanan latar belakang) SSH telah aktif (berwarna hijau `active (running)`).

systemctl status sshd
3

Koneksi dari Laptop (Host)

Buka CMD/Terminal di Laptop Anda. Lakukan *dial-up* menggunakan Username dan IP Server.

ssh admin@192.168.1.10
